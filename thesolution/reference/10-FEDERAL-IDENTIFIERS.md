# Federal Contract Identifier Reference

Authoritative reference for identifiers used across SAM.gov, FPDS, and USASpending.gov.

---

## Quick Reference

| Identifier | Length | Example | Purpose |
|---|---|---|---|
| **PIID** | 13-17 chars | `FA867810C0100` | Unique contract/award ID |
| **Modification Number** | 6 chars | `P00001` | Contract change (supplementary PIID) |
| **Solicitation Number** | 13-17 chars | `FA8678-09-R-1234` | Pre-award procurement notice ID |
| **NoticeId** | 32-char hex | `5b345bbb7127b91a3ad577b203fc6f68` | SAM.gov opportunity posting ID |
| **UEI** | 12 chars | `CTKLDK3V4M97` | Unique entity/company ID |
| **CAGE Code** | 5 chars | `4PF93` | Contractor facility ID |
| **NAICS Code** | 6 digits | `336411` | Industry classification |
| **PSC Code** | 4 chars | `1550` | Product/service category |
| **generated_unique_award_id** | variable | `CONT_AWD_FA867810C0100_9700_-NONE-_-NONE-` | USASpending award key |

## PIID (Procurement Instrument Identifier)

**Authority:** [FAR 4.1603](https://www.acquisition.gov/far/4.1603) / [DFARS 204.16](https://www.acquisition.gov/dfars/subpart-204.16-uniform-procurementinstrument-identifiers)

| Positions | Content | Example |
|---|---|---|
| 1-6 | Activity Address Code (AAC) | `FA8678` |
| 7-8 | Fiscal Year (last 2 digits) | `10` (FY2010) |
| 9 | Instrument Type Code | `C` |
| 10+ | Serial Number | `0100` |

- **DoD**: Always 13 characters (4-char serial). **Civilian**: 13-17 chars (4-8 char serial).
- Letters **I** and **O** prohibited everywhere (too similar to 1 and 0).
- Position 9 has 18+ valid codes (C=contract, D=IDV vehicle, F=task/delivery order, P=purchase order, R=RFP, Q=RFQ, etc.). **D vs F is key**: D = the indefinite-delivery vehicle, F = the order placed against it.
- Full instrument type code list: [FAR 4.1603(a)](https://www.acquisition.gov/far/4.1603)

---

## Modification Number

**Authority:** [FAR 4.1603(b)(2)](https://www.acquisition.gov/far/4.1603)

A 6-position suffix appended to the base PIID. First character: **P** = procuring office, **A** = admin office (e.g., DCMA). Positions 2-6 are sequential.

- The prefix indicates **who issued** it, not the type of change.
- Reason-for-modification codes (option exercise, funding, termination, etc.) are tracked separately in FPDS: [Reason for Modification codes](https://www.fpds.gov/help/Reason_for_Modification.htm)

---

## Solicitation Number

Same PIID structure (office + FY + type + serial). Type code at position 9 indicates pre-award instrument (R=RFP, Q=RFQ, B=IFB). The solicitation number and the eventual contract PIID are **different identifiers** -- no guaranteed structural link. Dashes in display formats are cosmetic.

---

## NoticeId (SAM.gov Opportunity ID)

- 32-character lowercase hex string (not a standard UUID -- no hyphens, don't assume RFC 4122).
- **Not the same as `opportunityId`** -- that term is third-party only.
- Multiple notices share the same `solicitationNumber` (original + amendments each get their own `noticeId`).
- No `GET /opportunities/{id}` endpoint; use search with `noticeid` filter.

---

## UEI (Unique Entity Identifier)

**Managed by:** GSA via SAM.gov. Replaced DUNS on **April 4, 2022**.

- Exactly 12 characters: uppercase A-Z + digits 0-9, **excluding O and I**.
- First character cannot be zero.
- Generated by SAM.gov during entity registration (no longer Dun & Bradstreet).
- Docs: [GSA UEI Specs](https://www.gsa.gov/about-us/organization/federal-acquisition-service/fas-initiatives/integrated-award-environment/iae-systems-information-kit/uei-technical-specifications-and-api-information), [Entity API](https://open.gsa.gov/api/entity-api/)

---

## CAGE Code (Commercial and Government Entity)

**Assigned by:** Defense Logistics Agency (DLA)

- Always 5 characters. Positions 1 & 5 must be numeric; positions 2-4 alphanumeric (excluding I, O).
- **CAGE** = U.S. entities. **NCAGE** = foreign (NATO). Both 5 chars.
- Independent from UEI -- both assigned during SAM registration but serve different purposes.
- Docs: [DLA CAGE Info](https://www.dla.mil/Working-With-DLA/Applications/Details/Article/2920893/cage-code-commercial-and-government-entity-code/)

---

## NAICS (North American Industry Classification System)

**Maintained by:** Census Bureau. **Current version:** NAICS 2022 (revised every 5 years).

- Hierarchical 2-through-6 digit system (sector -> subsector -> group -> industry -> national).
- **Federal contracts use the 6-digit code.** SBA assigns a size standard (max revenue or employees) to each.
- Docs: [Census NAICS](https://www.census.gov/naics/), [SBA Size Standards](https://www.sba.gov/federal-contracting/contracting-guide/size-standards)

---

## PSC (Product Service Codes)

**Maintained by:** GSA. Always 4 characters, first character = broad category.

- Docs: [PSC Manual](https://www.acquisition.gov/psc-manual)

## USASpending Composite Keys

### generated_unique_award_id

Format: `{prefix}_{PIID}_{agencyCGAC}_{parentPIID}_{parentAgency}`

| Segment | Example | Notes |
|---|---|---|
| Prefix | `CONT_AWD`, `CONT_IDV`, `ASST_AWD` | Contract, IDV vehicle, or assistance |
| PIID | `FA867810C0100` | The award's PIID |
| Agency | `9700` | 3-4 digit CGAC code ([full list](https://files.usaspending.gov/reference_data/agency_codes.csv)) |
| Parent PIID | `-NONE-` | Parent IDV PIID, or `-NONE-` if standalone |
| Parent agency | `-NONE-` | Parent IDV agency, or `-NONE-` if standalone |

PIID alone is **not globally unique** -- two agencies could theoretically issue the same PIID. The PIID + agency code combination is unique.

## Data Flow

```
Agency Contract Writing Systems
      |
      v
FPDS / SAM.gov Contract Awards
(FPDS.gov decommissioned Feb 2026; data entry now at sam.gov/contracting)
      |
      +---> SAM.gov (public award search)
      v
DATA BROKER (Treasury/OMB)  -->  USASpending.gov (~2 day lag)
```

- **SAM.gov has two separate domains:** (1) Contract Opportunities (solicitations) and (2) Contract Award Data (formerly FPDS). Distinct systems, shared domain.
- **Opportunity data does NOT flow to USASpending.** USASpending = awards/spending only.

---

## Record Hierarchy

Two parallel tracks converge at award:

**Pre-Award** (SAM.gov Opportunities): Notice (`noticeId`) -> amendments share same `solicitationNumber` -> links to eventual award (imperfect, no formal FK).

**Award** (FPDS/USASpending): Base award (PIID, mod 0) -> Modifications (P00001, A00001...) -> each mod = one FPDS transaction. USASpending rolls up all transactions into one "Award."

**Parent-Child** (IDV contracts): Parent IDV (own PIID) -> child orders (separate PIIDs referencing parent) -> each child has own modifications.
