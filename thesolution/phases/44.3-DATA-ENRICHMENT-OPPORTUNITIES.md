## Batch 1: High Value, Low Effort (2-3 days)

### 1.1 POC Data Loading -- TABLES EXIST, DATA NOT LOADED

**Impact: CRITICAL for business development**

`contracting_officer` (0 rows) and `opportunity_poc` (0 rows) tables exist but are empty. SAM.gov API returns full POC data on every opportunity:
```json
"pointOfContact": [
  {"type": "primary", "fullName": "Anthony Plath", "email": "anthony.plath@usmc.mil", "phone": ""},
  {"type": "secondary", "fullName": "Eric G. Daniels", "email": "Eric.G.Daniels@usmc.mil"}
]
```

**Fix**: Add POC extraction to `opportunity_loader._normalize_opportunity()`. Upsert into `contracting_officer` (dedup by name+email), link via `opportunity_poc`.
**Effort**: ~0.5 days. Schema exists.

### 1.2 Awardee Location + CAGE Code -- IN API, NOT STORED

For awarded opportunities, SAM.gov returns awardee location and CAGE code:
```json
"award": {"awardee": {"cageCode": "7ERY6", "location": {"city": {"name": "Huntsville"}, "state": {"code": "AL"}, "zip": "35805"}}}
```

We store `awardee_uei` and `awardee_name` but not location or CAGE.

**Fix**: Add columns: `awardee_cage_code VARCHAR(10)`, `awardee_city VARCHAR(100)`, `awardee_state VARCHAR(6)`, `awardee_zip VARCHAR(10)`. Update loader.
**Effort**: ~0.25 days.

### 1.3 Full Parent Path Storage -- CURRENTLY PARSED LOSSILY

We parse `fullParentPathName` ("DEPT OF DEFENSE.DEPT OF THE NAVY.USMC.MARCOR SYSCOM.COMMANDER") into `department_name`, `sub_tier`, `office` -- losing middle segments. The code path is also stored but only the last segment is kept.

**Fix**: Add `full_parent_path_name VARCHAR(500)` and `full_parent_path_code VARCHAR(200)`. Store raw values alongside parsed fields.
**Effort**: ~0.25 days.

### 1.4 Reference Table JOINs in C# API -- DATA EXISTS, NOT JOINED

Reference tables are loaded and rich, but the C# API services don't consistently JOIN them:

| Table | Code Column | Ref Table | What's Missing |
|-------|------------|-----------|----------------|
| fpds_contract.naics_code | raw code | ref_naics_code | No description in award results |
| fpds_contract.psc_code | raw code | ref_psc_code | No description in award results |
| fpds_contract.set_aside_type | raw code | ref_set_aside_type | No description/category |
| entity_business_type | raw code | ref_business_type | No description in entity detail |
| entity_naics | raw code | ref_naics_code | No description in entity detail |
| entity_psc | raw code | ref_psc_code | No description in entity detail |

**Note**: OpportunityService already JOINs ref_naics_code and ref_sba_size_standard for search results. Awards and entities don't.

**Fix**: Add LEFT JOINs in AwardService and EntityService.
**Effort**: ~0.5 days (multiple small joins).

## Batch 2: Medium Effort, High Value (2-3 days)

### 2.1 Vendor/Recipient Entity FKs -- UNLOCK FULL PROFILES

Three tables have vendor UEIs but no FK to entity (793K records):

- `fpds_contract.vendor_uei` -- no FK, no join
- `usaspending_award.recipient_uei` -- no FK, no join
- `sam_subaward.sub_uei` -- no FK, no join

**Fix**: Add FK constraints + service JOINs. Unlocks full vendor profile (addresses, NAICS, SBA certs, business types) in all award contexts.
**Effort**: ~0.5 days.

### 2.2 Contracting Office to Federal Hierarchy

`opportunity.contracting_office_id` is populated (14,240 of 14,241 rows) but never joined to `federal_organization`. The codes may need mapping investigation.

**Fix**: Investigate code matching (contracting_office_id vs fh_org_id vs agency_code). Add FK if clean match exists.
**Effort**: ~0.5 days investigation + 0.25 days implementation.

### 2.3 Period of Performance + Estimated Value Investigation

Both columns exist but are always NULL. Need to verify if SAM.gov API ever returns these fields.

**Fix**: Check raw staging data (`stg_opportunity_raw.raw_json`) for these fields across different opportunity types (presol, combined, award notice). If present for some types, add to loader.
**Effort**: ~0.25 days investigation.

### 2.4 State/Country Name Resolution

Multiple tables store state/country codes without names. Reference tables exist:

- `entity_address.state_or_province` -- no join to `ref_state_code`
- `entity_address.country_code` -- no join to `ref_country_code`
- `fpds_contract.pop_state/pop_country` -- no join
- `usaspending_award.pop_state/pop_country` -- no join

**Fix**: Add LEFT JOINs in service layer to resolve codes to names.
**Effort**: ~0.25 days.

## Batch 3: Cross-Entity Intelligence (3-5 days)

### 3.1 Opportunity to Award Chain

`opportunity.award_number` exists but has no FK to `fpds_contract.contract_id`. Can't easily trace "this opportunity was awarded as this contract."

**Fix**: Validate data quality, add FK if clean match, build service method.
**Effort**: ~0.5 days.

### 3.2 Award to Subaward Chain

`sam_subaward.prime_piid` has no FK to `fpds_contract.contract_id`. Can't trace prime/sub relationships.

**Fix**: Add FK + service JOINs for supply chain analysis.
**Effort**: ~0.5 days.

### 3.3 Opportunity Relationship Population

`opportunity_relationship` table exists (types: RFI_TO_RFP, PRESOL_TO_SOL, SOL_TO_AWARD) but has 0 rows. Not populated by the loader.

**Fix**: Detect related notices during loading (matching solicitation numbers across types). Populate relationship table.
**Effort**: ~1 day.

### 3.4 Eligible Vendor Query

No service method to answer "show me all WOSB companies in NAICS 541512 in Virginia" for a given opportunity. Requires joining opportunity's set_aside_code + naics_code against entity_naics + entity_sba_certification + entity_business_type.

**Fix**: Build `EntityService.SearchByOpportunity(noticeId)`.
**Effort**: ~1 day. Core to the prospecting use case.

### 3.5 Description Text Caching

`description_url` stores a URL; text fetched on-demand. Could cache for full-text search.

**Fix**: Add `description_text TEXT` column. Fetch during ETL (no rate limit on description endpoint).
**Effort**: ~0.5 days.
