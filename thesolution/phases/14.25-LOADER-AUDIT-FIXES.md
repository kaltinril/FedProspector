# Phase 14.25: Vendor API Loader Audit Fixes

**Status**: COMPLETE
**Created**: 2026-03-03
**Triggered by**: Full audit of all 6 vendor API loaders against API specs + website docs

---

## Summary

Comprehensive audit found 21 issues across all 6 vendor API loaders — wrong field mappings, silently dropped data, broken test fixtures, and column width problems. This phase fixes all HIGH and MEDIUM issues.

---

## HIGH Priority Fixes

### Awards Loader (`fed_prospector/etl/awards_loader.py`)

- [x] **H1**: Fix `extent_competed` — reads `solicitationProcedures.code` instead of `extentCompeted.code`. Change to `competition_info.get("extentCompeted", {}).get("code")`.
- [x] **H2**: Fix `far1102_exception_code/name` — hardcoded to `None`. Read from `awardee_data.get("far41102Exception") or {}` → `.get("code")` and `.get("name")`.
- [x] **H3**: Fix test fixture `tests/fixtures/sam_awards_response.json` — uses wrong JSON structure. Must match real API schema (nested `coreData`/`awardDetails`/`contractId`).

### Exclusions Loader (`fed_prospector/etl/exclusions_loader.py`)

- [x] **H4**: Add `is_fascsa_order` column to `sam_exclusion` (CHAR(3)), map from `exclusionOtherInformation.isFASCSAOrder`.
- [x] **H5**: Add `classification_type` column to `sam_exclusion` (VARCHAR(50)), map from `exclusionDetails.classificationType`.
- [x] **H6**: Fix test fixture `tests/fixtures/sam_exclusions_response.json` — uses v1 flat structure. Must use v4 nested structure (`exclusionDetails`, `exclusionIdentification`, `exclusionActions.listOfActions`).
- [x] **H7**: Fix `_extract_staging_key` — reads `raw.get("exclusionType")` but v4 nests it under `exclusionDetails`. Change to `raw.get("exclusionDetails", {}).get("exclusionType")`.

### Federal Hierarchy Loader (`fed_prospector/etl/fedhier_loader.py`)

- [x] **H8**: Fix `description` — hardcoded to `None`. Change to `raw.get("description")`.
- [x] **H9**: Fix `fhorgtype` case sensitivity — use `.lower()` or case-insensitive lookup in `_ORG_TYPE_LEVELS` to prevent `level=None`.
- [x] **H10**: Fix test fixture `tests/fixtures/sam_fedhier_response.json` — uses flat `cgac` key. Must use `cgaclist: [{cgac: "..."}]` array structure.

### Entity DAT Bulk Loader (`fed_prospector/etl/dat_parser.py`, `bulk_loader.py`)

- [x] **H11**: Fix SBA cert parsing in `_parse_sba_string` — split 10-char token into `sba_type_code = entry[:2]` and `certification_exit_date = _norm_date(entry[2:10])`. This is critical for WOSB/8(a) cert queries.
- [x] **H12**: Fix incremental bulk load — either implement proper upsert logic (REPLACE INTO or INSERT...ON DUPLICATE KEY UPDATE) or raise `NotImplementedError` explicitly instead of crashing on duplicate PK.

### CALC+ Loader (`fed_prospector/etl/calc_loader.py`)

- [x] **H13**: Remove `hourly_rate_year1` from `_API_FIELD_MAP` and `_RATE_COLUMNS`. Drop column from DDL and prod/test DBs (always NULL in v3 API).

### Opportunities Loader (`fed_prospector/etl/opportunity_loader.py`)

- [x] **H14**: Rename `description` column to `description_url` (VARCHAR(500)). Update loader, DDL, C# models/DTOs, and any views that reference it. Add comment that this is a URL to fetch the actual description text.
