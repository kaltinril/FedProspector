# Phase 44.8: View Efficiency Fixes

**Status:** COMPLETE
**Date:** 2026-03-07

## Problem

MySQL specialist review identified two views with aggregation fan-out bugs that produce **incorrect dollar amounts**:

1. **`v_competitor_analysis`** (20_competitor_analysis.sql) — Joins `fpds_contract` alongside `entity_business_type` and `entity_sba_certification` before GROUP BY. An entity with M business types, N SBA certs, and P contracts produces M x N x P intermediate rows, inflating `SUM(dollars_obligated)` by M x N.

2. **`v_incumbent_profile`** (40_incumbent_profile.sql) — Same pattern. Joins `entity_sba_certification` and `fpds_contract` before aggregating. `SUM(dollars_obligated)` is multiplied by the number of matching certifications.

## Fix

Pre-aggregate `fpds_contract` in a derived table (subquery) before joining, eliminating the Cartesian fan-out between detail tables.
