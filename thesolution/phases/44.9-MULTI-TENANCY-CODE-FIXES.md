# Phase 44.9: Multi-Tenancy Code Fixes

**Status**: PLANNED
**Priority**: Medium — Python CLI path broken, C# API path is correct
**Discovered**: Phase 44.6 validation audit

## Problem Statement

Phase 44.6 fixed the `prospect` unique key from `UNIQUE(notice_id)` to `UNIQUE(organization_id, notice_id)` for proper multi-tenant isolation. The C# API handles this correctly, but the Python `ProspectManager` does not include `organization_id` in its INSERT. Additionally, a minor DDL/C# default mismatch exists on `organization.profile_completed`.

## Fixes

### 1. Python `ProspectManager.create_prospect()` missing `organization_id`

The Python CLI path (`python main.py prospect create`) does not pass `organization_id` in its INSERT into the `prospect` table. The column is NOT NULL, so the command will fail.

**Files to change:**
- `fed_prospector/etl/prospect_manager.py` — Add `organization_id` parameter to `create_prospect()`, include in INSERT
- `fed_prospector/cli/prospecting.py` — Pass `organization_id` from CLI (add `--org` option or resolve from context)
- `fed_prospector/tests/test_prospect_manager.py` — Update tests for new parameter
- `fed_prospector/tests/test_cli_prospect.py` — Update CLI tests

### 2. DDL default mismatch: `organization.profile_completed`

- **DDL** (`60_prospecting.sql`): `profile_completed VARCHAR(1) NOT NULL DEFAULT ''`
- **C# entity** (`Organization.cs`): defaults to `"N"`
- **Fix**: Change DDL default to `'N'` and ALTER the live DB column default to match
