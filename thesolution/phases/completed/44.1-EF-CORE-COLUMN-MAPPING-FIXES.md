# Phase 44.1: EF Core Column Mapping Fixes

**Status**: PLANNED
**Priority**: Bug fix (blocks award detail page)
**Estimated Effort**: Small (2 files, ~10 lines changed)

## Problem

Navigating to an award detail page in the UI triggers two API errors, both caused by mismatches between what EF Core expects and what the DB/SQL actually provides.

### Error 1: `average_value` not found in `FromSql` (market-share endpoint)

The project uses `EFCore.NamingConventions` with `UseSnakeCaseNamingConvention()`. When `SqlQueryRaw<MarketShareDto>` runs, EF applies the snake_case convention to the DTO property names — so it looks for a column named `average_value` in the result set. But the SQL query uses PascalCase aliases (`AS AverageValue`), so EF can't find the expected snake_case columns.

### Error 2: `far1102exception_code` unknown column (award detail endpoint)

The snake_case naming convention converts `Far1102ExceptionCode` → `far1102exception_code` (digits don't count as word boundaries). But the actual DB column is `far1102_exception_code` (with underscore before `exception`).

## Fix

### 1. Change SQL aliases to snake_case in `GetMarketShareAsync`

**File**: `api/src/FedProspector.Infrastructure/Services/AwardService.cs` (lines 383-397)

Change SQL aliases to match what EF expects after snake_case convention mapping:
- `AS VendorName` → `AS vendor_name`
- `AS VendorUei` → `AS vendor_uei`
- `AS AwardCount` → `AS award_count`
- `AS TotalValue` → `AS total_value`
- `AS AverageValue` → `AS average_value`
- `AS LastAwardDate` → `AS last_award_date`

### 2. Add `[Column]` attributes to `FpdsContract` model

**File**: `api/src/FedProspector.Core/Models/FpdsContract.cs` (lines 97-101)

Add explicit `[Column]` attributes to override the naming convention:
```csharp
[Column("far1102_exception_code")]
[MaxLength(2)]
public string? Far1102ExceptionCode { get; set; }

[Column("far1102_exception_name")]
[MaxLength(100)]
public string? Far1102ExceptionName { get; set; }
```
