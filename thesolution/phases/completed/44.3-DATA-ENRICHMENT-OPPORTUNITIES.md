## Batch 1: High Value, Low Effort (2-3 days)

### 1.1 POC Data Loading -- COMPLETE

**Impact: CRITICAL for business development**

`contracting_officer` (0 rows) and `opportunity_poc` (0 rows) tables exist but are empty. SAM.gov API returns full POC data on every opportunity:
```json
"pointOfContact": [
  {"type": "primary", "fullName": "Anthony Plath", "email": "anthony.plath@usmc.mil", "phone": ""},
  {"type": "secondary", "fullName": "Eric G. Daniels", "email": "Eric.G.Daniels@usmc.mil"}
]
```

**Fix**: Add POC extraction to `opportunity_loader._normalize_opportunity()`. Upsert into `contracting_officer` (dedup by name+email), link via `opportunity_poc`.
**Note**: Dedup requires atomic upsert — same person across multiple opportunities should share one `contracting_officer` row. Use INSERT ... ON DUPLICATE KEY UPDATE on a unique index over (full_name, email).
**Effort**: ~0.5 days. Schema exists.

### 1.2 Awardee Location + CAGE Code -- COMPLETE

For awarded opportunities, SAM.gov returns awardee location and CAGE code:
```json
"award": {"awardee": {"cageCode": "7ERY6", "location": {"city": {"name": "Huntsville"}, "state": {"code": "AL"}, "zip": "35805"}}}
```

We store `awardee_uei` and `awardee_name` but not location or CAGE.

**Fix**: Add columns: `awardee_cage_code VARCHAR(10)`, `awardee_city VARCHAR(100)`, `awardee_state VARCHAR(6)`, `awardee_zip VARCHAR(10)`. Update loader.
**Effort**: ~0.25 days.

### 1.3 Full Parent Path Storage -- COMPLETE

We parse `fullParentPathName` ("DEPT OF DEFENSE.DEPT OF THE NAVY.USMC.MARCOR SYSCOM.COMMANDER") into `department_name`, `sub_tier`, `office` -- losing middle segments. The code path is also stored but only the last segment is kept.

**Fix**: Add `full_parent_path_name VARCHAR(500)` and `full_parent_path_code VARCHAR(200)`. Store raw values alongside parsed fields.
**Effort**: ~0.25 days.

### 1.4 Reference Table JOINs in C# API -- COMPLETE

**Effort**: ~0.5 days (multiple small joins).

### 2.2 Contracting Office to Federal Hierarchy -- DEFERRED TO PHASE 500

`opportunity.contracting_office_id` stores Level 3 (Office) FPDS codes like `36C786`, `W912DW`. `federal_organization` only has 908 rows — 170 departments (Level 1) and 738 sub-tiers (Level 2). Zero offices loaded.

**Investigation findings**:
- Match rate against all federal_organization columns: 0% (fh_org_id), 1.2% (agency_code, coincidental), 0.8% (oldfpds_office_code)
- The flat `/orgs` endpoint does NOT return offices — not even by direct `fhorgid` lookup
- The hierarchy endpoint (`/org/hierarchy?fhorgid=<sub_tier_id>`) DOES return office children with `fhorgtype: "OFFICE"`
- Verified: GSA Sub-Tier "Federal Acquisition Service" (fhorgid=300000583) returns 9 office children
- Example office: fhorgid=100179081, name="4730-10FZT", agencycode=4730, code path="4700.4730.10FZT"
- The FOUO (For Official Use Only) API returns offices in the flat endpoint, but requires government credentials
- No bulk download exists for office-level data

**Implementation plan** (when API quota allows):
1. Add `get_children(fhorgid)` method to `SAMFedHierClient` using `/org/hierarchy` endpoint
2. Add `load hierarchy --offices` CLI command that iterates all 738 sub-tiers and collects office children
3. Resumable: track which sub-tiers have been processed, skip on re-run
4. Initial load: ~738 API calls (one per sub-tier). Future updates: only re-check sub-tiers with newer `lastupdateddate`
5. Existing `_normalize_organization()` already handles `fhorgtype: "OFFICE"` via case-insensitive lookup
6. After offices loaded, match `opportunity.contracting_office_id` via code path last segment

**Status**: Deferred to Phase 500. Requires ~738 API calls (one per sub-tier) using the hierarchy endpoint. Need a day with full API quota on key 2.

### 2.3 Period of Performance + Estimated Value -- DEFERRED TO PHASE 500

Both columns exist but are always NULL. DDL comments mark these as "Reserved: not in public Opportunities API v2 response." These fields are not available to non-government API consumers. Period of performance IS available on awarded contracts via `fpds_contract.period_of_performance_start/end` — it's only the pre-award opportunity stage where SAM.gov withholds it. Sites like GovWin (Deltek) get this data through special agreements, FPDS-NG feeds, and manual research — not the public API.

**Status**: Dead end for opportunities via SAM.gov public API. Moved to Phase 500.

### 2.4 State/Country Name Resolution -- COMPLETE

Multiple tables store state/country codes without names. Reference tables exist:

- `entity_address.state_or_province` -- no join to `ref_state_code`
- `entity_address.country_code` -- no join to `ref_country_code`
- `fpds_contract.pop_state/pop_country` -- no join
- `usaspending_award.pop_state/pop_country` -- no join

**Fix**: Add LEFT JOINs in service layer to resolve codes to names.
**Effort**: ~0.25 days.
