# Phase 14.8 ‚Äî Architecture Compliance & Missing CLI Search Commands

## Status: COMPLETE

## Context

The system has a strict data-flow rule:

```
Vendor APIs ‚Üí raw/staging tables ‚Üí ETL ‚Üí local DB tables
                                             ‚Üì
                                      C# API (58 endpoints) ‚Üí UI
                                             ‚Üì
                              main.py search/analyze (local DB only)
```

**`main.py load *`** commands are the only things that may call vendor APIs.
**`main.py search *`** commands must query the local MySQL database exclusively.

An audit (Phase 14.8) found three violations and two gaps in this architecture, all in the Python CLI. The C# API layer (13 controllers, 58 endpoints) was found fully compliant ‚Äî zero external API calls.

---

## Issues Fixed

### 1. `search exclusions` was calling the SAM.gov Exclusions API (BUG)

**File**: `fed_prospector/cli/exclusions.py`

The `check_exclusion` function (registered as `search exclusions` in main.py) was instantiating `SAMExclusionsClient` and querying the live SAM.gov API on every invocation. It only fell back to the local `sam_exclusion` table when the daily API quota was exhausted ‚Äî exactly backwards from the correct behavior.

**Fix**: Removed all API client code from `check_exclusion`. The function now calls `_check_local_only(uei, name)` directly. Removed the `--key` option (no longer needed). Updated docstring and output messages to remove "(local DB)" parenthetical (it's always local now) and updated the stale-data hint to use new CLI group syntax.

### 2. `search entities` command was missing (GAP)

**File**: `fed_prospector/cli/entities.py`

The `search` CLI group had coverage for opportunities, agencies, subawards, and exclusions ‚Äî but not entities/vendors. There was no way to query the locally loaded entity data via CLI even though the `entity`, `entity_naics`, `entity_business_type`, and `entity_sba_certification` tables are fully populated by the ETL pipeline.

**Fix**: Added `search_entities` command (registered as `search entities`) with filters:
- `--uei` ‚Äî exact match on UEI
- `--name` ‚Äî partial match on legal business name
- `--naics` ‚Äî filter by NAICS code (EXISTS on `entity_naics`)
- `--state` ‚Äî filter by state code (EXISTS on `entity_address`)
- `--cert` ‚Äî filter by SBA certification type: WOSB, 8A, HUBZone, SDVOSB (EXISTS on `entity_sba_certification`, active certs only)
- `--active-only` ‚Äî filter to `registration_status = 'A'`
- `--limit` ‚Äî default 25

Output: tabular with UEI, name, CAGE, state, primary NAICS, registration expiration.

### 3. `search awards` command was missing (GAP)

**File**: `fed_prospector/cli/awards.py`

No way to query the locally loaded `fpds_contract` table from the CLI. The `search` group had no awards command.

**Fix**: Added `search_awards` command (registered as `search awards`) with filters:
- `--naics` ‚Äî filter by NAICS code
- `--set-aside` ‚Äî filter by set-aside type (WOSB, 8A, etc.)
- `--agency` ‚Äî filter by agency ID (exact) or agency name (partial)
- `--vendor` ‚Äî filter by vendor UEI (exact) or vendor name (partial)
- `--piid` ‚Äî filter by contract PIID (exact; shows all modifications)
- `--from-date` / `--to-date` ‚Äî filter by `date_signed`
- `--limit` ‚Äî default 25

Defaults to `modification_number = '0'` (base awards only) to avoid showing the same contract multiple times. Suppressed when `--piid` is specified.

Output: tabular with PIID, vendor name, dollar amount, NAICS, set-aside, agency, date signed, number of offers.

## Phase 14.8 Part 2: API Terminology Audit

A documentation audit was conducted across CLAUDE.md, QUICKSTART.md, MASTER-PLAN.md, and phase docs to identify ambiguous uses of the word "API".

### Terminology Definitions

| Term | Definition |
|------|-----------|
| **Vendor API** | External government data sources: SAM.gov (v1‚Äìv4), USASpending.gov, GSA CALC+. Called only by Python `load` commands. Rate-limited (SAM.gov key 2: 1,000/day). |
| **App API** | FedProspect's own C# ASP.NET Core backend. 58 endpoints, 13 controllers. Queries local MySQL only. Never calls Vendor APIs. Consumed by the React UI. |

See `thesolution/reference/06-GLOSSARY.md` (API Types section) for the canonical reference.

### Audit Findings

Across 6 docs: 42 clearly-Vendor occurrences, 23 clearly-App occurrences, **2 ambiguous** (fixed below).
