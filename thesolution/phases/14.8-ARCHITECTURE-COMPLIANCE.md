# Phase 14.8 — Architecture Compliance & Missing CLI Search Commands

## Status: COMPLETE

## Context

The system has a strict data-flow rule:

```
Vendor APIs → raw/staging tables → ETL → local DB tables
                                             ↓
                                      C# API (57 endpoints) → UI
                                             ↓
                              main.py search/analyze (local DB only)
```

**`main.py load *`** commands are the only things that may call vendor APIs.
**`main.py search *`** commands must query the local MySQL database exclusively.

An audit (Phase 14.8) found three violations and two gaps in this architecture, all in the Python CLI. The C# API layer (14 controllers, 57 endpoints) was found fully compliant — zero external API calls.

---

## Issues Fixed

### 1. `search exclusions` was calling the SAM.gov Exclusions API (BUG)

**File**: `fed_prospector/cli/exclusions.py`

The `check_exclusion` function (registered as `search exclusions` in main.py) was instantiating `SAMExclusionsClient` and querying the live SAM.gov API on every invocation. It only fell back to the local `sam_exclusion` table when the daily API quota was exhausted — exactly backwards from the correct behavior.

**Fix**: Removed all API client code from `check_exclusion`. The function now calls `_check_local_only(uei, name)` directly. Removed the `--key` option (no longer needed). Updated docstring and output messages to remove "(local DB)" parenthetical (it's always local now) and updated the stale-data hint to use new CLI group syntax.

### 2. `search entities` command was missing (GAP)

**File**: `fed_prospector/cli/entities.py`

The `search` CLI group had coverage for opportunities, agencies, subawards, and exclusions — but not entities/vendors. There was no way to query the locally loaded entity data via CLI even though the `entity`, `entity_naics`, `entity_business_type`, and `entity_sba_certification` tables are fully populated by the ETL pipeline.

**Fix**: Added `search_entities` command (registered as `search entities`) with filters:
- `--uei` — exact match on UEI
- `--name` — partial match on legal business name
- `--naics` — filter by NAICS code (EXISTS on `entity_naics`)
- `--state` — filter by state code (EXISTS on `entity_address`)
- `--cert` — filter by SBA certification type: WOSB, 8A, HUBZone, SDVOSB (EXISTS on `entity_sba_certification`, active certs only)
- `--active-only` — filter to `registration_status = 'A'`
- `--limit` — default 25

Output: tabular with UEI, name, CAGE, state, primary NAICS, registration expiration.

### 3. `search awards` command was missing (GAP)

**File**: `fed_prospector/cli/awards.py`

No way to query the locally loaded `fpds_contract` table from the CLI. The `search` group had no awards command.

**Fix**: Added `search_awards` command (registered as `search awards`) with filters:
- `--naics` — filter by NAICS code
- `--set-aside` — filter by set-aside type (WOSB, 8A, etc.)
- `--agency` — filter by agency ID (exact) or agency name (partial)
- `--vendor` — filter by vendor UEI (exact) or vendor name (partial)
- `--piid` — filter by contract PIID (exact; shows all modifications)
- `--from-date` / `--to-date` — filter by `date_signed`
- `--limit` — default 25

Defaults to `modification_number = '0'` (base awards only) to avoid showing the same contract multiple times. Suppressed when `--piid` is specified.

Output: tabular with PIID, vendor name, dollar amount, NAICS, set-aside, agency, date signed, number of offers.
